Markdown 教程

什么是 Markdown

Markdown 是一种轻量级标记语言，创始人为约翰·格鲁伯（John Gruber）。它允许人们使用易读易写的纯文本格式编写文档，然后转换成有效的 XHTML（或者HTML）文档。

简单的来说： * Markdown是一种语言 * 这种语言可以简单的写一些标记了一定格式的文本形式的文档 * 这种语言可以比较方便的转换成Html类的文档，可以用于制作静态网页。

为什么会学习Markdown

说实话，我想了半个月了，完全没有想起来我当时为什么会去学习Markdown，只能确定是在我疯狂的想做一个个人的博客或者说是主页的那段时间。我觉得最有可能得原因是在当时的一众的wiki框架、博客技术帖中，在某一天我把"wiki"和"python"两个关键词放在一起进行了搜索，然后找到了"mkdocs"这个python包（一个可以把Markdown文档变成静态网页的工具），就顺带学了Markdown了。

而正好在后续的学习中，突然发现这个这个东西还挺有用的，不仅在Github中可以用来写ReadMe文档来介绍自己的项目，还能快乐的在简书、CSDN等国内平台上直接来发表一些帖子，甚至可以通过一些插件直接导出word文档。

换句话说，Markdown文档写好了之后就可以像"底片"一样"转印"到各种格式的文档、平台之中，这个是我觉得很重要的一点。如果说，我用word辛辛苦苦码出了一篇文章，当我想把他放在我的博客或者其地方时，我就需要重新对它进行排版，因为它的标题、分段大概率不会被保留，即使保留了，在一些平台也可能发生错误，或者根本不被认可。但是如果我是用Markdown写的，那它的这些格式都像字符一样在文章中体现着，只要我写下了"# XXX"，那么它永远就是一个一级标题，无论我把它放在哪里，它都不会发生变化。所以我不断地学习Markdown语言，并写下了这篇文章，既作为我个人的一个笔记，也用来帮助一些可能想学习Markdown的人。

Markdown的优缺点

优点： * 可以简单的转译到各个各个平台，在任何设备上看起来都很棒，而不会发生排版上的缺失。 * 使用简单 * 目前相关的插件多

缺点： * 不同平台的兼容性可能不一样。有些平台可以接受某些标记，而某些就不行。 * 它不会做任何花哨的事情，比如改变字体大小、颜色或类型（不过这个好想可以通过添加html语言来帮忙解决）

（关于优缺点这块我觉得在我以后实际使用的时候可以补充不少）

详细介绍

"标题"大概率是每个人最先用到的东西，毕竟每篇文章都要有一个标题作为开端，在Markdown里，标题的写法很简单，几级标题就在行首输入几个井字符（“#”），然后输入一个 空格 ,然后在输入自己想要的标题就OK，比如我想以“Markdown教程”作为一级标题，那么我只需要从行首输入1个井字符，1个空格，以及“Markdown教程”，在回车就ok。

# “Markdown教程”

为了保证索引目录比较好看，标题就不在这里演示了，大致效果可以看上边的“各级标题示意图”

作为一级和二级标题一般是比较重要的，比如在mkdocs建立目录索引的过程中，会自动为一级二级目录建立索引，而在vscode中，也只有一级二级标题下面加一条横线，就看着比较醒目。

在书写上，一级二级目录也有着不同的写法，可以先键入标题文字，然后换行，键入一个或两个“=”（一级标题）或者“-”（二级标题），比如我想输入一个“Markdown教程”的一级标题和一个“开始学习”的二级标题，我就可以按照一下方式输入

Markdown教程
=
开始学习
-

或者是

Markdown教程
==
开始学习
--

它们的输出效果和用井字符的输出效果是一样的。

详细介绍

按照中文的书写习惯，缩进肯定是必不可少的，不过在Markdown的语言中没有明确的用来段首缩进的符号，不过如果为了书写好看，可以使用html中的 、 、 、 、 、 等空格标记实现缩进。在使用的时候，要特别注意两点。第一点是要记得打 分号 ！！！第二点是这几个空格标记所标记的空格大小是不同的，分别是四分之一汉字、二分之一汉字和一个汉字，也就是如果实现一般中文写作习惯中的空两个字符位置，就分别要输入8个、4个、2个。

效果：

详细介绍

换行也是常用操作之一，既可以使用在行位连续输入两个空格实现换行，也可以使用连续两次的回车实现换行。个人在编辑的时候建议后者，因为在大部分编辑器的时候空格并不明显，如果在某个应该换行地方少打了一个空格，不容易查出来，而使用两次回车的换行既方便检查，又能让文本的格式更加的清晰。

斜体、粗体和粗斜体

代码及效果

'''斜体'''
*文字内容*
_文字内容_

'''粗体'''
**文字内容**
__文字内容__

'''粗斜体'''
***文字内容***
___文字内容___

效果：

斜体

粗体

粗斜体

详细介绍

斜体、粗体以及粗斜体是文章中常用的用来提示重点的方法，在Markdown中，都可以通过使用星字符（“*”）或下划线（“_”）完成，两种符号显示效果相同。斜体、粗体以及粗斜体分别需要输入1个、2个和3个星字符 （“*”）或下划线（“_”）后输入需要修饰的文本，然后再次输入同样数量的符号将文本内容包含起来即可。

如*斜体*、**粗体**以及***粗斜体***即可完成对于文本的修饰。（下划线同理）不过在日常使用中建议使用“*”，因为“*”比“_”更容易数清输入的个数，从而更好的检查文档修饰的正确性。

分割线

代码及效果

'''以下是三条分割线'''
***
---
___

效果：

分割线示意图

详细介绍

分割线可能是我们平常写作中用到不多的一种修饰符，不过在网页上它还挺重要的，它可以快速的对于区域进行分块。其输入方法也很简单，只需要在本行内单独输入“*”、“-”或“_”连续3个及以上就行，是的，输入三个就能达到分割线的效果，不过为了使文档编译前可读性高一点或者单纯发泄一下，可以输入满满一行的“-”（如果不输入换行符的话一直都是一行哦~）。
