Markdown 教程

什么是 Markdown

Markdown 是一种轻量级标记语言，创始人为约翰·格鲁伯（John Gruber）。它允许人们使用易读易写的纯文本格式编写文档，然后转换成有效的 XHTML（或者HTML）文档。

简单的来说：

  • Markdown是一种语言
  • 这种语言可以简单的写一些标记了一定格式的文本形式的文档
  • 这种语言可以比较方便的转换成Html类的文档，可以用于制作静态网页。

为什么会学习Markdown

说实话，我想了半个月了，完全没有想起来我当时为什么会去学习Markdown，只能确定是在我疯狂的想做一个个人的博客或者说是主页的那段时间。我觉得最有可能得原因是在当时的一众的wiki框架、博客技术帖中，在某一天我把"wiki"和"python"两个关键词放在一起进行了搜索，然后找到了"mkdocs"这个python包（一个可以把Markdown文档变成静态网页的工具），就顺带学了Markdown了。

而正好在后续的学习中，突然发现这个这个东西还挺有用的，不仅在Github中可以用来写ReadMe文档来介绍自己的项目，还能快乐的在简书、CSDN等国内平台上直接来发表一些帖子，甚至可以通过一些插件直接导出word文档。

换句话说，Markdown文档写好了之后就可以像"底片"一样"转印"到各种格式的文档、平台之中，这个是我觉得很重要的一点。如果说，我用word辛辛苦苦码出了一篇文章，当我想把他放在我的博客或者其地方时，我就需要重新对它进行排版，因为它的标题、分段大概率不会被保留，即使保留了，在一些平台也可能发生错误，或者根本不被认可。但是如果我是用Markdown写的，那它的这些格式都像字符一样在文章中体现着，只要我写下了"# XXX"，那么它永远就是一个一级标题，无论我把它放在哪里，它都不会发生变化。所以我不断地学习Markdown语言，并写下了这篇文章，既作为我个人的一个笔记，也用来帮助一些可能想学习Markdown的人。

Markdown的优缺点

优点：

  • 可以简单的转译到各个各个平台，在任何设备上看起来都很棒，而不会发生排版上的缺失。
  • 使用简单
  • 目前相关的插件多

缺点：

  • 不同平台的兼容性可能不一样。有些平台可以接受某些标记，而某些就不行。
  • 它不会做任何花哨的事情，比如改变字体大小、颜色或类型（不过这个好想可以通过添加html语言来帮忙解决）
  • 在很多时候空格会引发错误，与普通的书写习惯相悖。（这点很重要，不要轻易的输入空格）

（关于优缺点这块我觉得在我以后实际使用的时候可以补充不少）

详细介绍

斜体、粗体以及粗斜体是文章中常用的用来提示重点的方法，在Markdown中，都可以通过使用星字符（“*”）或下划线（“_”）完成，两种符号显示效果相同。斜体、粗体以及粗斜体分别需要输入1个、2个和3个星字符 （“*”）或下划线（“_”）后输入需要修饰的文本，然后再次输入同样数量的符号将文本内容包含起来即可。

如*斜体*、**粗体**以及***粗斜体***即可完成对于文本的修饰。（下划线同理）不过在日常使用中建议使用“*”，因为“*”比“_”更容易数清输入的个数，从而更好的检查文档修饰的正确性。

脚注

代码及效果

'''脚注'''
文字内容 [^脚注符号]
（空行）
[^脚注符号]: 脚注内容

效果：

文字内容 ^脚注符号

详细介绍

脚注是在论文、科普性文章中很重要的一个功能，来源的引用都需要用到脚注功能。同时它也是折磨了我和我的学习团队很久的一个功能，我们在学习它的时候出现了很多的意外情况，接下来我将详细介绍。

首先，如果想添加一个脚注，只需要在文章需要添加脚注的地方输入“ [\^] ”，并在“^”后面输入上脚注的名称或者符号即可，然后在文章之后，换行，输入同样的“ [\^脚注名称] ”，并输入一个英文的冒号（“:”）之后输入脚注的内容即可。

脚注示例

但是！切记，首先，在正常文本与脚注内容中一定要有空行。其次，在脚注内容中，一定不要有空格！如果需要键入空格，请用\&nbsp\;或者其他空格符号（参见缩进），否则将造成错误，不能正确识别为脚注。

详细介绍

在word中，编号和项目符号功能是一个很常用的功能。当然，编号功能在word里也是一个很让人头疼的功能，它常常不受作者的控制随机产生各种顺序的编号，好在在Markdown中作者对于文档的控制能力强多了。在Markdown中，这两个个功能被称为列表。

列表分为有需列表和无序列表，其中无序列表的实现仅需要在行首输入星字符（“”）十字符（“+”）或短横线（“-”）其中1个1次，然后输入1个 空格* ，之后输入本列内的文本内容即可。

而有序列表需要输入一个数字，一个英文句号后输入1个 空格 ，之后输入本列内的文本内容即可。有趣的是，即使你输入的数字并不是按序的，在编译的时候可能会帮你自动按出现顺序拍好序，不过建议为了文档的可读性，还是按序输入数字。一件有趣的事，我并不知道如何重新开启一个新的列表，换句话说我并不知道如何重新编号，在上面所展示的效果中，我是在“多级列表”前加了一个分割线实现的重新编号，刚刚查了百度也没有给我明确的答案。

多级列表嵌套的实现主要通过缩进实现，在列表前输入4个空格或者一个"tab"即可将该列表降级成为次一级的列表，在不同的平台上，不同的次级列表有不同的标志。

详细介绍

在写科技类博客的时候，代码区块是一个很重要的部分，之前我也没有很好的研究过这一部分，只使用过制表符（“tab”）来写代码，这次也借这个机会好好的了解了一下再Markdown中不懂代码区块之间的区别。

代码区块有三种实现方法，分别是使用制表符（“tab”），1个反引号（“`”，键盘数字键“1”左边那个键）和3个反引号。这三种符号都比较好理解，观察一下上面代码的举例基本就可以知道怎么使用，所以我就不在这赘述它们的使用方法了，主要介绍一下它们之间的区别。

首先是1个反引号所形成的代码区块，这种方式的代码区块仅仅能生成一行代码，即使在文档中给它换行了在最后显示的时候它还是一行代码，所以这种方式我觉得更适合用在于说明单独一个命令的时候，比如hello world，不过它也有自己的优点，如你所见，它可以直接嵌套在文档的文字说明之中，而不用单独产生一个区块，这是3个反引号与制表符所办不到的。

而3个反引号与制表符最大的优势是可以生成一个区块，在这个区块里可以比较自由的实现换行，这样可以更好的写作一段代码。它们之间的不同是在不同的平台中对于这两个代码区块的编译可能不同，比如在vscode中，3个反引号可以选择输入的代码是某种语言的，从而根据该种语言实现关键词的高亮，但是在mkdocs中，仅对制表符编写的代码区块实现了高亮，而对于3个反引号的代码区块没有对应反应。不过作为中文作者，输入3个反引号就意味着势必要切换一次输入法，所以制表符是在写作时成本最低的一种输入方式。

链接

代码及效果

'''链接'''
[链接名称](链接地址)

<链接地址>

[链接名称][链接变量]

[链接变量]:链接地址

效果： 百度

https://www.baidu.com

百度

详细介绍

Markdown的常见链接有两种插入方式，分别是使用“[链接名称](链接地址)”和“\<链接地址>”。前者会在显示的时候保留“链接名称”并进行高亮处理，然后隐藏“链接地址”，后者则是直接显示“链接地址”并进行高亮。需要注意的是，在使用后一种方式的时候，链接地址需要写上协议名比如百度需要写“http://www.baidu.com”而不能仅仅写www.baidu.com。

然后还有一种高级的链接方式，它可以在“链接名称”后面设定一个“链接变量”，形如“[链接名称][链接变量]”，然后在文章的末尾通过“[链接变量]:链接地址”的形式填充变量，这种方式比较方便在文章中多处设定相同的链接跳转后统一维护。

详细介绍

在Markdown表格中，使用竖线“|”来分隔单元格，使用短横线“-”分隔表头和其他行。如果希望该列表格左对齐就在短横线的左侧添加一个冒号“:”，形成一个形如“:-”的标记，该列表格将会全部左对齐，右对齐同理，在短横线右侧添加冒号，形如“-:”，即可使该列表格全部右对齐。如果想要居中，则需要再短横线两侧分别添加冒号，形如“:-:”即可使该列表格全部居中显示。其中短横先可以输入1个及以上，显示效果相同，不过可以增加文档的可读性。（“:-:”与“:---------:”效果相同）

特别提示，这个左右对齐以及居中对于表头并不产生影响，如果一定要居中需要使用html语言。

支持HTML元素部分

在菜鸟教程中，所提出的Markdown支持的html的标签有kbd、b、i、em、sup、sub、br等。

释义：

  • \<kbd>——键盘标签
  • \<b>——加粗
  • \<i>——斜体
  • \<em>——强调（在不同的平台会显示成斜体或者粗体）
  • \<sup>——上标
  • \<sub>——下标
  • \<br>——换行（感觉这个东西在Markdown里换行还挺好使的，我之后试试）
